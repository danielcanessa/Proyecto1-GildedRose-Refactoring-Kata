# Práctica Guiada: Refactorización con Principios de Construcción

Esta práctica forma parte del Proyecto de Construcción. Del 20%, este ejercicio vale 10%.

**Repositorio sugerido:** [GildedRose Refactoring Kata](https://github.com/emilybache/GildedRose-Refactoring-Kata)

## Objetivo

Aplicar principios como DRY, SRP, KISS, bajo acoplamiento, etc. a código legado.

## Instrucciones

1. Tomar del repositorio las siguientes clases:
   - [GildedRose.java](https://github.com/emilybache/GildedRose-Refactoring-Kata/blob/main/Java/src/main/java/com/gildedrose/GildedRose.java)
   - [Item.java](https://github.com/emilybache/GildedRose-Refactoring-Kata/blob/main/Java/src/main/java/com/gildedrose/Item.java)

   O el paquete:

   - [Java](https://github.com/emilybache/GildedRose-Refactoring-Kata/tree/main/Java)

2. Subir el paquete completo como un proyecto de Java, o las clases a un proyecto de Java, al repositorio creado como tarea en GitHub.
3. Identificar malas prácticas en las clases `GildedRose.java` e `Item.java`.
4. Modificar el código para aplicar principios como modularidad, separación de responsabilidades, DRY y KISS, cohesión, desacoplamiento, legibilidad y mantenibilidad, reutilización de componentes y SOLID.
5. Subir los cambios con commits claros y justificados.
6. Documentar los cambios y decisiones en `README.md`.
7. Subir el paquete o los códigos fuente a la tarea, o subir un archivo con la dirección del repositorio que contiene el código.
8. **3% de puntos extra:** aplicar el patrón de comportamiento Strategy, explicado en la clase.

## Enfoque

Esta práctica debe orientarse a resolver problemas reales de mantenimiento y escalabilidad, fomentando el pensamiento crítico y la toma de decisiones sobre qué refactorizar primero y por qué.
